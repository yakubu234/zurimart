1. Project Identity

 1. Public Retailer: No login required; buys 1–49 loaves at standard prices.
 2. Internal Outlet/Minimart: Authorized accounts for branch-to-bakery restocking.

 5. Super Admin: Full visibility of financials, inventory, and branch performance.

4. Key Functional Modules

A. Smart Order Routing & Production "Handshake"
To prevent production conflicts, the system uses an interactive tagging system:
 • Outlet Tagging: Outlets select/tag a specific production branch when placing an order.

 • Available: Ready for new orders.
 • Overly Booked: Reached capacity for the selected date; hides branch from selection.

 • Demand Indicators: Outlets must toggle if a request is for Wholesale Demand (to fulfill a large external client) or Retail Demand (to restock the minimart).
